Intro to Linux

Linux is a lightweight operating system

It can run on low power devices to supercomputers

Presently, Linux runs on 95% of web servers and 85% of smartphones (Android is built on top of the Linux kernel)

***The reason that Linux is such a flexible operating system is that almost all of its features can be accessed using a text terminal, without a graphical desktop environment - which gives me faster access to a computer or server’s resources***

 

 

 

 

 

 

 

 

Unix History

Linux itself was not created until 1991-92, however, the roots of it began as UNIX back in 1969

UNIX was proprietary software

in 1983 Richard Stallman started the GNU project with the goal of creating open-source software

 

 

 

 

 

 

 

 

Open Source Software

Open source software is software with source code that anybody can look at or contribute to and distribute based on the terms of the license

open source is "copyleft" instead of copyright, meaning that any derivative software must be distributed under the same or equivalent license terms

Linux itself is an example of open-source software

anyone in the world can contribute to open source software as long as they adhere to the "rules" of the software itself

 

 

 

 

 

 

 

 

OS vs Linux Kernel vs Shell

An operating system has three main components:

Graphical Desktop aka GUI - the portion the end user interacts with most, shows the graphical reprensentation of files, folders, etc

Shell - the interface between the user and kernel, a command line interpreter where I or an end user can enter commands to perform tasks

Kernel - controls all the major functions of the hardware, such as process management, task scheduling and managing hardware devices

 

Linux interprets commands from the bash command language

Input

input is giving a computer a command or commands to execute, be it code itself, a keystroke on a keyboard, or a click of a mouse

before anything, a connection must be established from the input device (monitor, keyboard, mouse, speakers, bluetooth devices, etc) to the computer, through either a physical port or a bluetooth port

the input device "reports" events (mouse clicks, keystrokes, etc) to the CPU by using binary code (1s and 0s)

once the input is converted into binary, it is sent to the CPU to be processed

 

 

 

Processing

the job of processing information is given to the CPU - central processing unit

this is the "brain" of the computer

the CPU controls all the different components between hardware and software, and has the responsibility of establishing communication between them

 

 

 

Memory

Memory, in the context of computers, refers to the system or device used to store computer-based data temporarily or permanently

threre are 2 types of memory - Primary and Secondary memory

Primary Memory aka RAM is used by the CPU to retrieve data that needs to be accessed immediately or temporarily

When a command to run a program is sent to the CPU, the CPU retrieves data from RAM to access what instructions it needs to execute

data on RAM is lost if it is not saved to the Hard Drive of the computer - aka Secondary Memory

 

secondary memory is a hard drive within the computer or an external / removable hard drive

 

 

 

 

Output

After the CPU processes data and sends out instructions on how to handle it, output is produced

Some output devices are monitors, printers, speakers

output is the final step of computer interaction

The Linux Filesystem

Linux OS's have a structure that separates directories needed/accessible to the end user from directories needed by the administrator

Linux follows the FHS - Filesystem Hierarchy Standard

Linux uses a tree structure and the root directory ( / ) is the starting point for the entire Linux filesystem

 

any directory that is a child of the root directory is a sub-directory

all of these sub-directories serve a purpose

 

Linux and Windows / MacOS are similar, but have key differences

Windows uses separate data drives like C:\ and D:\ whereas Linux branches everything off from /

 

MacOS has an almost identical structure but adds sub-directories that Linux does not have:

Documentation can be accessed from the CLI using these files and commands:

/usr/share/doc/ directory - contains README files and subdirectories with README files

man command - gives a brief synopsis of the program, description, etc

info command - gives a full of the program, description, etc

 

 

Compression / Archive Utilities

Linux has multiple file compression types:

gzip - retains the original file’s ownership modes, access, and modification timestamps, compressed files have the .gz extension

bzip2 - compressed files have the .bz2 extension

xz - compressed files have the .xz extension

the -k option keeps the original file and does not allow for it to be overridden

the -d command decompresses files

the -r command compresses all of the files in a directory

ZIP and TAR allow for compression and archiving of files, whereas the other 3 do not

even though zip is more popular, tar is preferred across Linux distros because it stores Unix file attribures, retains files permissions and other metadata

Types of Linux Users

Linux is designed from the ground up to prevent individual users from accessing areas of the file structure they shouldn’t access

Linux environment administrators have to define separate accounts and permissions to determine who can access specific files, directories, applications, and resources

Generally speaking, Linux users are 1 of 3 kinds:

 

administrator - superuser AKA root user, has full control over entire system

normal user - employees / real people, have defined access to system resources

system user - non human or computer generated account, performs specfic tasks, has defined access

 

Users and Groups in Linux

All users added are assigned a name, unique user ID (UID), group, and group id (GID)

the UIDs and GIDs have a numbering system starting with 0

the id and groups commands check the current user's UID and GID

 

File Permissions in Linux

in Linux, EVERYTHING is based on file permissions

Each file or directory has an owner and a group (or groups) that usually has more permissions to read, write, or execute than users not in the owner or in the permission group

every file / directory has a permission line that explains eveything about the file / directory

 

each letter permission can be written as a number, and the permissions are grouped in threes

Directories are set to 755 - files within that directory are set to 644

Elevating Privileges (sudo)

the sudo can be given to give a logged in non-administrator account TEMPORARY priviledges to do admin things
