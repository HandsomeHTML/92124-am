Role & Function of the vi Text Editor

there are 2 versions of vi - vi and vim

the vi command runs the vi command

the vim command runs vim - some Linux distros need it to be installed

vi itself has 4 modes:

 

normal mode - allows for moving around in the editor since vi was made prior to mice and arrow keys

command line mode - allows for searching, saving and quitting - can quit out of vi 4 different ways

insert mode - allows for inserting, appending or creating new lines above/below the current line

search mode - allows for finding a word, phrase or string

 

Normal mode uses the following keystrokes for movement:

h - mvoes the cursor one character to the left

l - mvoes the cursor one character to the right

k - mvoes the cursor one character to the line above

j - mvoes the cursor one character to the line below

CTRL-D - moves the cursor down half a page

CTRL-U - moves the cursor up half a page

CTRL-B - moves the cursor BACK a full page

CTRL-F - moves the cursor FORWARD a full page

 

w - moves cursor right to the NEXT word

b - moves cursor left to the PREVIOUS word

^ - moves cursor to the beginning of the line

$ - moves cursor to the last character on the line

nG - moves the cursor to line n

gg - moves the cursor to the FIRST line of the document

G - moves the cursor to the LAST line of the document

What is a Shell?

the shell is a program that functions as an user interface and command line interpreter when using a command line interface

Linux provides 2 types of user interfaces:

CLI and GUI

 

every time a program is started, Linux makes a process for it, and gives it a unique process iD (PID)

a process is a single instace of a program that operates in its own memory space

 

there are 2 types a of processes - parent and child

when a command is executed from a CLI, a child process (subshell) is created to engage the OS - the child process gets its own PID

 

There are 2 types of variables - shell and environment

a variable is a memory location that is assigned a name and is used to store data

a shell variable only exists in the memory space of the shell in which it was created

shell variable syntax is < variable_name >=< value >

to view the contents of a variable, echo $ < variable_name >

the set command displays a list of all the shell variables and functions in the current process

environment variables are different from shell variables - environment variables are visible to child processes

shell variables can be converted to environment variables by assigning an export attribute to the variable

< variable > or export < variable >=< value > are the syntax that can be used

the unset command deletes the variable and removes it from the environment

the env command displays a list of all environment variables in the current shell

 

Configuring Aliases

alias is a command shortcut - that exists only in the shell in which it was created or loaded

to remove an alias from memory, the command is unalias (alias name)

to PERMANENTLY remove an alias, the command is unalias

best practice is to check to see if an alias name is in use and that is done with type -a (proposed alias name)

the \ negates the special significance of the following character or word, so the command echo \$shell would just be "$shell" on output or if a alias changes a color, the \ makes the alias show the default color

 

Setting Up Local Environment

Locale settings are used to define output formats used by applications depending on the location of the Linux computer

locale setting can be viewed / set with the following:

locale -a < command > - shows the values of locale categories set

localectl < command > - view/modify system locale and keyboard settings

local categories are associated with a locale name - the locale name defines formatting for a specific region

the command locale -a or localectl list-locales lists available locale names

***localectl list-locales is important***

if changes are made, they are saved on the /etc/locale.conf file

changes to /etc/locale.conf file can be verified with localectl status

the command date is primarily used for reporting information from and managing the system clock

 

the time can be changed with date -s and using the format:

"Day Month Date Time Time-Zone Year"

   

the directory /usr/share/zoneinfo/ and the file /etc/localtime are used to configure the system’s time zone

there are 4 commands that can display the current system time zone:

date

date +%Z

timedatectl

ls -l /etc/localtime

 

the commands hwclock, hwclock -r or hwclock --show is used to manage the real-time clock, or RTC and display the current hardware time clock

the command hwclock -s or hwclock --hctosys (hardware clock to system clock) sets the system time using the RTC and sets the time zone to the current time zone

The command hwclock -w or hwclock --systohc sets the system clock to the same time as the hardware clock

 

The command timedatectl or timedatectl status can be used to view and manage system clocks

timedatectl list-timezones displays a list of time zones

timedatectl set-timezone changes the system-wide time zone

timedatectl set-time 'YYYY-MM_DD HH:MM' changes BOTH the system time and date

   

Bash Configuration Files, Redirection

there are 5 Bash Configuration Files:

 

/etc/profile

/etc/bashrc

/home/< username >/.bash_profile

/home/< username >/.bashrc

/home/< username >/.bash_logout

 

/etc/profile and /etc/bashrc are system-wide config files applied to ALL applicable users when they log in

files in a user's home directory have config info for that user ONLY

/etc/profile and /home/< username >/.bash_profile contain commands to configure the user's working environment

/etc/bashrc and /home/< username >/.bashrc are used to configure Bash shell features like aliases

/home/< username >/.bash_logout is used during a user being logged out

 

Linux has a login script order when my user's log in:

/etc/profile

/home/< username >/.bash_profile

/home/< username >/.bashrc

/etc/bashrc

the source command makes the configuration changes available immediately

 

Redirection

when commands are executed, the input is received through the standard input device - value 0 (stdin) and the output (except for error msgs, stderr - value 2) standard output device (stdout) value -1 device

files are accessed by the kernel using a file descriptor and a file descriptor number is assigned to a file when it is opened by a process

a file descriptor table is created for every process

 

< redirects standard input

heredoc is another form of redirection - it reads each input line from the current source until it reaches the limit word

> redirects standard output to a file, and >> appends data to a file

2> and 2>> are control operators and redirect error msgs to a file or device

1> redirects standard output messages and is the same as >

stderr and stdout can be combined - for example, 2>&1 will redirect the standard error to the same location as the standard output

1>&2 would redirect the standard output to the same location as the standard error

2> /dev/null ignores error msg displayed to the screen

pipe is a way to connect the stdout of one process to the stdin of another process using the | symbol

the tee command displays the output of a command to the console device and redirects the output to a file or device

Understanding Linux Users and Groups

Linux uses a method called Discretionary Access Control (DAC) to permit or restrict access to files

On DAC-based systems the account owner decides who has permission to access, delete, or modify their files

a user is a person or service that requires access to system files or resources

a user account is a method of providing / restricting access to system resources

linux implements user ID ranges (UID) to organize users - depending on the UID, the user many have few or many priviledges

the administrative account is called ROOT and can do ANYTHING in the computer

**best practice to ONLY have 1 root account***

administrative users have a UID range of 0-99 - any user assigned the user ID 0 has root priviledges

the admin accounts are added to the operating system during the installation process

Linux abides by the "least priviledge" principle - meaning user accounts should only have enough access to do their job and nothing more

User account ID ranges are specified by the variables UID_MIN and UID_MAX in /etc/login.defs - the file that defines new user defaults, such as the location of the user’s mailbox, their user ID, and their password aging defaults

the default minimum UID is 1000 and the deafult maximum UID is 60000

Linux services or system accounts can be assigned a user account and UIDs between 0-99, or between the value set by SYS_UID_MIN and SYS_UID_MAX in /etc/login.defs

useradd --system < system account name > creates a system account


Unlike user accounts, system accounts cannot log on to the system, do not require a password, do not have password aging applied, and do not have home directories



Information for users / groups are stored in the following configuration files:

/etc/passwd - Contains user account information

/etc/shadow - Contains user password and password aging information, only root user(s) can see

/etc/group - Contains a list of groups and their members



/etc/passwd has 1 line of a unique user record, and seven fields:

User_Name - supplied by the admin

password - stored in /etc/shadow, only the root user(s) can see

UID - Linux kernel identifies a user by their user ID, not their username

GID - group ID number of user's primary group - a user can be a member of multiple groups - when a user creates a file, the primary group ID is assigned as the file's group owner **secondary group memberships are found in /etc/groups

Comment - by default, has the user's full name

Home_directory contains absolute path to users home directory - on login, they are placed at this directory

default shell - contains absolute path of shell interpreter, most Linux systems default to bash but some can default to /bin/ksh (Korn shell) or /bin/csh (C shell)


system accounts do not require a default shell, and have either /sbin/nologin or /bin/false in the absolute path field


each record in /etc/passwd should have a corresponding record in /etc/shadow

the shadow file has 8 fields:

Username - user login name

Password - encrypted password, if value is !! then the password has never been set

Last Modified- # of days since January 1, 1970 that the password was last changed

Min_Days - minimum # of days required before a password can be changed - default value is specified in /etc/login.defs

Max_days- maximum number of days before a password expires - default value is specified in /etc/login.defs

Warn_Days - # of days prior to password expiration the user will be warned of the pending expiration - default value is specified in /etc/login.defs

Inactive- # of days after password expiration the user account will be disabled. The purpose of this field is to prevent open accounts that are not being used - During the period between password expiration and the number of inactive days exceeded, the user may still log on but will be forced to change their password. After the number of inactive days is exceeded, the account is disabled and requires an administrator to remediate the situation. The default INACTIVE value is specified in /etc/default/useradd.

Expire - This field displays the number of days since January 1, 1970, after which the account will be disabled



group information is stored in the /etc/group and /etc/gshadow files - the /etc/gshadow file maintains the list of encrypted group passwords
